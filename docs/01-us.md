## US001 — Adicionar item ao carrinho

### Identificação
Projeto: EBAC-SHOP  
Versão afetada: 1.0  
Tipo: História  
Prioridade: Média  
Pontuação: 13  

---

### Descrição
Como cliente da EBAC-SHOP  
Quero adicionar produtos ao carrinho  
Para realizar a compra dos itens  

---

### Regras de Negócio

- Não é permitido inserir mais de 10 itens de um mesmo produto ao carrinho;
- O valor total do carrinho não pode ultrapassar R$ 990,00;
- Valores entre R$ 200,00 e R$ 600,00 ganham desconto de 10%;
- Valores acima de R$ 600,00 ganham desconto de 15%.

---

### Critérios Funcionais Esperados

- O sistema deve permitir adicionar produto ao carrinho;
- O sistema deve calcular corretamente o subtotal;
- O sistema deve aplicar automaticamente a regra de desconto conforme o valor total;
- O sistema deve impedir a adição de mais de 10 unidades do mesmo produto;
- O sistema deve impedir que o total ultrapasse R$ 990,00.

---

### Observação de execução no projeto

Durante a execução automatizada no ambiente disponibilizado para o projeto, foram identificadas divergências entre parte das regras esperadas e o comportamento real da aplicação, especialmente nos cenários relacionados aos descontos por faixa de valor e ao limite máximo do carrinho.

Esses comportamentos foram mantidos no projeto como evidências de não conformidade observadas durante a execução dos testes automatizados.

---

Durante a execução automatizada no ambiente disponibilizado para o projeto, foi identificada divergência no comportamento esperado para bloqueio após 3 tentativas inválidas de login.

## US003 — API de Cupons

### Identificação
Projeto: EBAC-SHOP  
Versão afetada: 1.0  
Tipo: História  
Prioridade: Média  
Pontuação: 13  

---

### Descrição
Como administrador da EBAC-SHOP  
Quero utilizar o serviço de cupons  
Para listar e cadastrar cupons no sistema  

---

### Autenticação da API

- Usuário: admin_ebac  
- Senha: @admin!&b@c!2022  
- Authorization: Basic YWRtaW5fZWJhYzpAYWRtaW4hJmJAYyEyMDIy  

---

### Regras de Negócio

#### GET
- Deve listar todos os cupons cadastrados;
- Deve permitir buscar cupom por ID.

#### POST
- Deve cadastrar cupons com os campos obrigatórios:
  - code (ex: "Ganhe10")
  - amount ("10.00")
  - discount_type ("fixed_product")
  - description ("Cupom de teste")
- O código do cupom não pode ser repetido;
- Os demais campos são opcionais.

---

### Critérios Funcionais Esperados

- A API deve permitir listar os cupons cadastrados;
- A API deve permitir consultar um cupom por ID;
- A API deve permitir cadastrar um cupom com os campos obrigatórios preenchidos;
- A API não deve permitir cadastro de cupom com nome repetido.

## US004 — Catálogo de Produtos no aplicativo mobile

### Identificação
Projeto: EBAC-SHOP  
Versão afetada: 1.0  
Tipo: História  
Prioridade: Média  
Pontuação: 8  

---

### Descrição
Como cliente da EBAC-SHOP  
Quero visualizar o catálogo de produtos no aplicativo  
Para encontrar os itens que desejo comprar  

---

### Regras de Negócio

- O catálogo deve listar os produtos disponíveis para compra;
- O usuário deve conseguir visualizar informações principais do produto, como nome, imagem e preço;
- O usuário deve conseguir navegar até a área de catálogo por meio da navegação principal do aplicativo;
- O usuário deve conseguir tentar acessar os detalhes de um produto ao selecioná-lo;
- Quando disponível no aplicativo, o usuário deve conseguir filtrar produtos por categoria.

---

### Critérios Funcionais Esperados

- O sistema deve permitir acesso à área de catálogo no aplicativo;
- O sistema deve exibir produtos disponíveis para navegação;
- O sistema deve permitir tentativa de acesso aos detalhes de um produto;
- O sistema deve manter a navegação funcional durante o fluxo principal;
- A validação automatizada deve considerar a estabilidade disponível no aplicativo fornecido.

---

### Observação de execução no projeto

A história relacionada ao catálogo mobile foi implementada com abordagem adaptada, em razão da instabilidade observada no aplicativo disponibilizado para o projeto.

Dessa forma, a cobertura foi mantida com foco em navegação principal, acesso ao catálogo, tentativa controlada de acesso aos detalhes do produto e manutenção da consistência técnica da automação dentro das condições disponíveis no ambiente de testes.
